Files and Descriptions


1. 01_create_tables.sql

Purpose:

Defines the structure of database tables using the CREATE TABLE statement.

Key features:

Creates relational database tables

Defines columns and appropriate data types

Establishes primary keys to uniquely identify records

Prepares the database schema for inserting and querying data

Example:

CREATE TABLE users (
    user_id SERIAL PRIMARY KEY,
    user_name VARCHAR(100),
    country VARCHAR(100),
    registration_date DATE
);

This step establishes the foundation of a relational database system.

2. 02_insert_into.sql

Purpose:

Demonstrates inserting records into tables using the INSERT INTO statement.

Key features:

Inserts single and multiple rows

Specifies column names and values

Supports various data types such as text, numbers, and dates

Example:

INSERT INTO users (user_name, country, registration_date)
VALUES ('Alice Smith', 'South Africa', '2024-01-15');

This operation simulates loading initial data into a database.

3. 03_select_queries.sql

Purpose:

Demonstrates retrieving data from tables using the SELECT statement.

Key features:

Retrieves specific columns or all columns

Uses column aliases for readability

Displays data stored within database tables

Example:

SELECT user_name, country
FROM users;

The SELECT statement is the most commonly used SQL command for retrieving data.

4. 04_filter_queries.sql

Purpose:

Introduces filtering results using the WHERE clause.

Key features:

Filters rows based on conditions

Supports comparison operators (=, >, <, >=, <=)

Retrieves targeted datasets

Example:

SELECT user_name, country
FROM users
WHERE country = 'South Africa';

Filtering is essential for narrowing down large datasets during analysis.

5. 05_and_or_conditions.sql

Purpose:

Demonstrates combining multiple filtering conditions using logical operators.

Key features:

Uses AND to require multiple conditions

Uses OR to allow alternative conditions

Demonstrates complex query filtering

Example:

SELECT user_name, country
FROM users
WHERE country = 'South Africa'
AND registration_date > '2024-01-01';

Logical conditions are commonly used when analyzing datasets with multiple constraints.

6. 06_order_by_queries.sql

Purpose:

Shows how to sort query results using the ORDER BY clause.

Key features:

Sorts results in ascending (ASC) or descending (DESC) order

Can sort by multiple columns

Helps organize datasets for reporting and analysis

Example:

SELECT user_name, registration_date
FROM users
ORDER BY registration_date DESC;

Sorting helps structure results when reviewing or presenting data.
