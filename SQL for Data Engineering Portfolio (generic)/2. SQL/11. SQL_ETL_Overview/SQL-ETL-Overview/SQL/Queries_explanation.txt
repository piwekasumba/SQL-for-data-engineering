Understanding ETL
1. Extract

The Extract phase involves retrieving data from source systems such as operational databases, APIs, CSV files, or application databases.

In SQL-based workflows, extraction often involves selecting data from one or more source tables.

Example:

SELECT *
FROM raw_transactions;

This query retrieves raw data from a source table that may require transformation before it is used for analysis.

2. Transform

The Transform phase involves cleaning, filtering, and restructuring the extracted data to make it suitable for analysis or storage in a data warehouse.

Transformations may include:

Filtering records

Converting data types

Aggregating values

Standardizing formats

Removing duplicate records

Example:

SELECT
    user_id,
    amount,
    transaction_date,
    amount * 1.15 AS amount_with_tax
FROM raw_transactions
WHERE amount > 0;

This transformation filters valid transactions and performs a calculation on the extracted data.

3. Load

The Load phase inserts the transformed data into a target table such as a reporting database, analytics dataset, or data warehouse.

Example:

INSERT INTO cleaned_transactions (user_id, amount, transaction_date)
SELECT
    user_id,
    amount,
    transaction_date
FROM raw_transactions
WHERE amount > 0;

This query loads cleaned data into a structured table designed for reporting or analytics.

Files and Descriptions
1. 01_extract_data.sql

Purpose:
Demonstrates extracting raw data from source tables using SELECT statements.

Key features:

Retrieves raw datasets from operational tables

Prepares source data for transformation

Simulates the extraction phase of an ETL pipeline

Example:

SELECT *
FROM raw_users;
2. 02_transform_data.sql

Purpose:
Demonstrates transforming extracted data before loading it into a target table.

Key features:

Filters invalid records

Performs calculations and data formatting

Standardizes values for consistent storage

Example:

SELECT
    user_id,
    LOWER(email) AS normalized_email,
    registration_date
FROM raw_users
WHERE email IS NOT NULL;
3. 03_load_data.sql

Purpose:
Demonstrates loading transformed data into destination tables.

Key features:

Inserts cleaned datasets into structured tables

Simulates loading into an analytics-ready database

Supports downstream data analysis

Example:

INSERT INTO users_cleaned (user_id, email, registration_date)
SELECT
    user_id,
    LOWER(email),
    registration_date
FROM raw_users;
Relevance to Data Engineering

ETL pipelines are one of the primary responsibilities of data engineers. Organizations rely on ETL processes to move data from operational systems into analytics platforms and data warehouses.

Data engineers use ETL processes to:

Consolidate data from multiple sources

Clean and standardize raw datasets

Prepare data for business intelligence and analytics

Maintain reliable and scalable data pipelines

SQL is often used within ETL pipelines for performing transformations and loading processed data into structured databases.
