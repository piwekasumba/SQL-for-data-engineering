Overview

This folder contains PostgreSQL SQL scripts and notes demonstrating analytical and window functions used in data engineering. Analytical functions perform calculations across a set of rows related to the current row, enabling advanced analysis without collapsing the dataset.

These functions are critical for:

Generating running totals and moving averages

Ranking and ordering data within partitions

Performing cumulative calculations

Producing insights for reporting and analytics

Key Analytical Functions

1. ROW_NUMBER()

Assigns a unique sequential integer to rows within a partition of a result set.

Example:

SELECT 
    user_id,
    country,
    ROW_NUMBER() OVER (PARTITION BY country ORDER BY registration_date) AS row_num
FROM users;

Use case: Identify the first or last record in each group.

2. RANK() and DENSE_RANK()

Assign ranks to rows within a partition.

RANK() skips numbers after ties.

DENSE_RANK() does not skip numbers after ties.

Example:

SELECT 
    user_id,
    amount,
    RANK() OVER (PARTITION BY country ORDER BY amount DESC) AS rank_amount
FROM transactions;

Use case: Rank top-performing users or transactions by value.

3. SUM() and AVG() as Window Functions

Calculates aggregate values across a defined window without collapsing rows.

Example:

SELECT 
    user_id,
    amount,
    SUM(amount) OVER (PARTITION BY country) AS total_country_amount,
    AVG(amount) OVER (PARTITION BY country) AS avg_country_amount
FROM transactions;

Use case: Analyze cumulative or average metrics within groups.

4. LEAD() and LAG()

Access subsequent or preceding rows in a result set, useful for trend analysis.

Example:

SELECT 
    user_id,
    transaction_date,
    amount,
    LAG(amount) OVER (PARTITION BY user_id ORDER BY transaction_date) AS previous_amount,
    LEAD(amount) OVER (PARTITION BY user_id ORDER BY transaction_date) AS next_amount
FROM transactions;

Use case: Compare current values with previous or next periods.

5. NTILE()

Divides rows into a specified number of buckets or tiles, often used for percentile calculations.

Example:

SELECT 
    user_id,
    amount,
    NTILE(4) OVER (ORDER BY amount DESC) AS quartile
FROM transactions;

Use case: Segment users or transactions into performance quartiles or deciles.
