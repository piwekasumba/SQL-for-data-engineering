📌 What the Data Represents

The dataset contains insurance claim records that reflect typical scenarios encountered in operational business environments. It includes details such as:

Column	Description
claim_id	Unique identifier for each claim
policy_id	Identifier for the insurance policy
customer_id	Identifier for the customer who filed the claim
claim_date	Date when the claim was submitted
claim_amount	Monetary amount claimed
claim_status	Status of the claim (e.g., Approved, Rejected, Pending)
policy_type	Type of insurance policy (e.g., Auto, Health, Property)

Note: Column names and types are chosen to represent realistic dataset structure used in analytical databases.

🧪 Example SQL Analytics Queries

Below are representative examples of SQL queries that demonstrate analytical capabilities:

-- Total claims per customer
SELECT customer_id,
       SUM(claim_amount) AS total_claimed
FROM insurance_claims
GROUP BY customer_id;

-- Average claim amount by policy type
SELECT policy_type,
       AVG(claim_amount) AS average_claim
FROM insurance_claims
GROUP BY policy_type;

-- Approved claims only
SELECT *
FROM insurance_claims
WHERE claim_status = 'Approved';

-- Claims submitted in 2025
SELECT *
FROM insurance_claims
WHERE claim_date >= '2025-01-01';

Each query has been tested against the dataset and documented in this repository’s SQL scripts to show clear analytical intent and outcome.
