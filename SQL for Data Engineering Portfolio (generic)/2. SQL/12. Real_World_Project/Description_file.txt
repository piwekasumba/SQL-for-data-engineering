Real-World SQL Data Engineering Project
1. Project Overview

This project is a hands-on SQL data engineering repository that demonstrates practical skills in working with relational databases, data manipulation, and ETL pipelines.

The project simulates a small-scale e-commerce dataset with tables for customers, products, and orders. The goal is to practice real-world data engineering workflows, including:

Creating and designing relational database schemas

Inserting, updating, and managing transactional data

Querying data for analysis and reporting

Debugging SQL errors and handling common mistakes

Building foundational ETL pipelines

This repository is designed to showcase skills relevant for entry-level data engineering positions in South Africa, focusing on PostgreSQL and SQL best practices.

2. Project Objectives

The primary objectives of this repository are:

Database Design & Modeling

Define tables with proper data types and relationships

Use primary and foreign keys to enforce data integrity

Demonstrate normalized schema design

SQL Data Manipulation

Insert, update, and delete records

Perform complex queries to extract business insights

Use aggregate functions, joins, subqueries, and filtering

Debugging and Error Handling

Identify and fix syntax errors and logical mistakes

Understand PostgreSQL error messages and solutions

Document common mistakes for learning and reference

ETL Pipeline Simulation

Extract data from source tables

Transform raw data using SQL queries

Load processed data into new summary tables for analytics

Documentation & Professional Practices

Provide clear, structured notes and explanations

Demonstrate industry-standard workflow practices

Prepare the repository for review by recruiters and hiring managers

3. Data Schema

The project includes the following tables:

Customers Table

Columns: customer_id, name, country

Stores customer information

Products Table

Columns: product_id, product_name, price

Stores product catalog information

Orders Table

Columns: order_id, customer_id, order_date, amount

Stores transactional order data and links to customers

The schema demonstrates relational table design and key constraints, similar to real-world systems.

4. Example Queries

The project demonstrates the following query types:

Basic SELECT statements

Filtering with WHERE

Sorting with ORDER BY

Aggregations with GROUP BY and HAVING

Joins (INNER JOIN, LEFT JOIN)

Subqueries and nested queries

ETL-style insertions into summary tables

5. Debugging & Best Practices

The repository also focuses on:

Identifying syntax and logical errors in SQL queries

Correcting column and table reference issues

Preventing unsafe updates or deletions

Applying best practices for efficient and readable SQL

This section emphasizes the engineering mindset, not just writing queries.

6. ETL Pipeline Simulation

ETL (Extract, Transform, Load) operations are demonstrated with:

Extracting raw transactional data from orders

Transforming data using aggregation and filtering

Loading processed data into analytical tables like customer_summary

This mimics real-world data pipelines commonly used in South African companies and international firms.
