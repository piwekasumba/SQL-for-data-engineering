Files and Descriptions


1. 01_insert_into.sql

Purpose:

Demonstrates how to insert records into database tables using the INSERT INTO statement.

Key features:

Inserts single rows and multiple rows

Specifies column names and corresponding values

Works with different data types such as text, numeric values, and timestamps

Example:

INSERT INTO users (user_name, country, email)
VALUES ('John Doe', 'South Africa', 'john@example.com');

This operation is commonly used when loading new data into a database system.

2. 02_update_records.sql

Purpose:

Demonstrates modifying existing records within a table using the UPDATE statement.

Key features:

Updates specific columns using the SET clause

Uses WHERE conditions to target particular rows

Helps maintain and correct stored data

Example:

UPDATE users
SET country = 'Namibia'
WHERE user_name = 'John Doe';

Updates are frequently used when correcting data or applying transformations.

3. 03_delete_records.sql

Purpose:

Illustrates removing data from tables using the DELETE statement.

Key features:

Removes records selectively using conditions

Demonstrates safe deletion practices

Prevents accidental deletion of entire tables by emphasizing WHERE clauses

Example:

DELETE FROM transactions
WHERE transaction_id = 101;

Deletion operations are used when cleaning datasets or removing outdated records.

4. 04_filter_queries.sql

Purpose:

Demonstrates retrieving filtered data using SELECT statements combined with WHERE conditions.

Key features:

Filters rows based on conditions

Uses logical operators such as AND and OR

Retrieves targeted subsets of data for analysis

Example:

SELECT user_name, country
FROM users
WHERE country = 'South Africa'
AND user_name IS NOT NULL;

Filtering is commonly used when preparing datasets for analytics and reporting.

5. 05_order_by_queries.sql

Purpose:
Shows how to sort query results using the ORDER BY clause.

Key features:

Sorts results in ascending (ASC) or descending (DESC) order

Can sort by multiple columns

Helps organize query output for reporting or analysis

Example:

SELECT user_name, registration_date
FROM users
ORDER BY registration_date DESC;

Sorting is often used to display ranked or chronological datasets.
